## 🔧 三大核心模块详解

### 1️⃣ 多智能体架构（Manus）- `multi_agent.py`

**四个角色分工**：

| Agent | 职责 | 特点 |
|-------|------|------|
| **Planner** | 分析任务，生成执行计划 | 输出有序子任务清单 |
| **Executor** (×N) | 并行执行子任务 | 独立上下文，不爆炸 |
| **Verifier** | 检查中间结果 | 发现问题通知重规划 |
| **Coordinator** | 异步协调管理 | 任务状态、依赖关系 |

**执行流程**：
```
任务 → Planner 分解 → 多 Executor 并行执行 → Verifier 验证
  ↓
  如果有问题 → Planner 重规划 → 继续执行
  ↓
  全部完成 → 返回结果
```

**优势**：
- ✅ 自动分解复杂任务
- ✅ 并行执行提升效率
- ✅ 独立上下文避免 token 爆炸
- ✅ 自动纠错机制

---

### 2️⃣ 分级推理引擎（GPT-5.5）- `adaptive_reasoning.py`

**五个推理等级**：

| 等级 | 耗时 | 适用场景 | 深度 |
|------|------|---------|------|
| **none** | <1s | 简单问候、查询 | 直接回答 |
| **low** | 1-3s | 轻度分析 | 初步思考 |
| **medium** | 3-10s | 中等复杂度 | 多路径分析 |
| **high** | 10-30s | 复杂问题 | 深度推理 |
| **xhigh** | 30-120s | 超复杂任务 | 完整思维链 |

**自动选择机制**：
- 任务长度 + 关键词 + 代码指标 → 复杂度评分 → 自动选级别

**内部验证**：
- high/xhigh 等级自动进行逻辑检查
- 发现问题自动修正

**优势**：
- ✅ 简单任务快速回答，节省 token
- ✅ 复杂任务深度推理，确保正确性
- ✅ 自动选择，无需手动调参
- ✅ 内部自我验证，错误率降低

---

### 3️⃣ 自进化循环（Hermes）- `self_evolving.py`

**三个核心机制**：

| 机制 | 作用 | 效果 |
|------|------|------|
| **向量记忆** | 历史经验向量化 + 检索 | 新问题自动找相似案例 |
| **技能自动生成** | 失败时分析缺陷 → 自造技能 | 能力边界自动扩张 |
| **经验学习** | 每次任务完成保存经验 | 越用越聪明 |

**执行流程**：
```
新任务来了
  ↓
向量检索：有没有类似的历史经验？
  ↓
如果有 → 直接使用历史方案（快速）
如果没有 → 正常执行
  ↓
任务完成 → 保存经验向量 + 提取通用技能
  ↓
下次遇到类似问题，自动检索到这个经验
```

**优势**：
- ✅ 历史经验复用，加快解决速度
- ✅ 自动生成新技能，能力不断扩张
- ✅ 越用越聪明，形成个性化知识库

## 💡 三大模块如何协同工作

### 场景 1：简单问题（用户问"今天几号"）

```
用户问题 → 自动选择 none 级别推理 → 直接回答 → 完成
                    (adaptive_reasoning)
耗时：<1 秒
```

### 场景 2：中等复杂问题（用户问"设计一个 Web 应用"）

```
用户问题 → 自动选择 medium 级别推理
  ↓
检查向量记忆：有没有类似的设计案例？
  ↓（有）使用历史方案 + 微调
  ↓（没有）正常推理
  ↓
完成后保存经验向量 + 提取通用技能
          (self_evolving)
耗时：3-10 秒
```

### 场景 3：超复杂问题（用户问"为我构建一个完整的 SaaS 产品"）

```
用户问题 → 自动选择 xhigh 级别推理
  ↓
Planner 分解成子任务：
  1. 需求分析
  2. 架构设计
  3. 前端开发
  4. 后端开发
  5. 部署方案
  ↓
多 Executor 并行执行每个子任务
  ↓
Verifier 逐个验证结果
  ↓
如果有问题 → Planner 重规划 → 继续执行
  ↓
全部完成 → 保存整个项目经验
          (self_evolving)
耗时：30-120 秒
```
