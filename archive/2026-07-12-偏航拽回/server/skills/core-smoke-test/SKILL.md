# core-smoke-test

## Contract
把“测试一下核心功能/随便做点什么”转成最小、可复现的烟雾测试；任何单项失败都继续测其余项，并把“工具可用”“功能正常”“结果为空”分开记录。

## Phases
1. 先确认当前环境和可用工具，列出本次要测的最小集合。
2. 依次执行低风险代表性动作：Shell、Python、临时文件读写、记忆/状态、联网或抓取（若可用）。
3. 用表格汇总 `功能/结果/证据/备注`，最后给出一句总评和下一步建议。

## 注意
- 只测代表性路径，不做无关扩展。
- 失败要写明是权限、网关、无结果还是程序异常。
- 不把“命令能跑”误写成“业务功能已正常”。
- 临时测试优先，避免改动真实数据。
