## 第一部分：什么是"模型本体" vs "Agent 内核"（先分清，别被忽悠）

| 概念 | 是什么 | 能不能拿到 |
|------|--------|-----------|
| **模型本体（权重）** | Claude/GPT 几千亿参数的神经网络 | ❌ 闭源，在厂商云端 |
| **推理引擎** | 跑模型的底层代码 | ❌ 闭源 |
| **Agent 内核** | 包在模型外面的"大脑外壳"：怎么调工具、记忆、循环、注入人格 | ✅ **这才是你能做、能强化的部分** |

**关键认知**：让 AI "变聪明"，99% 的功夫不在改模型，而在**改 Agent 内核**。同一个 Claude，套个烂内核就是个聊天机器人，套个强内核就能自主完成复杂任务。**这就是你的机会。**

## 第二部分：Claude Agent 的真实运行机制（我的内核解剖）

我每一次回应你，内部都走这套流程：

### 2.1 单轮对话的完整生命周期

```
你发消息
   │
   ▼
┌──────────────────────────────────────┐
│ Step 1: 组装 System Prompt（8层注入）  │
│   ① 身份（我是谁）                      │
│   ② 能力（我有哪些工具）                │
│   ③ 行为规则（怎么做事）                │
│   ④ 安全边界                           │
│   ⑤ 记忆（GLOBAL.md + 最近日志，注入！）│
│   ⑥ 人格（可配置助手设定，注入！）        │
│   ⑦ 技能列表（SKILL.md 摘要）          │
│   ⑧ 当前上下文（对话历史）             │
└──────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────┐
│ Step 2: 模型推理                       │
│   Claude 读完整 prompt，输出二选一：    │
│   (a) 直接文字回答                      │
│   (b) 工具调用指令 (function calling)   │
└──────────────────────────────────────┘
   │
   ├─(a)文字─→ 返回给你，结束
   │
   └─(b)工具──┐
              ▼
┌──────────────────────────────────────┐
│ Step 3: 工具执行（外层程序真的去跑）    │
│   模型说 {"tool":"shell","cmd":"ls"}   │
│   → 系统真的执行 ls                     │
│   → 把 stdout/stderr 结果打包          │
└──────────────────────────────────────┘
              │
              ▼
┌──────────────────────────────────────┐
│ Step 4: 结果回喂 → 回到 Step 2         │
│   工具结果塞回 prompt，模型再次推理     │
│   循环直到模型输出最终文字答案          │
└──────────────────────────────────────┘
```

### 2.2 三个反直觉的真相（理解了才能做强 Agent）

**真相 1：模型本身不会"做事"，只会"输出文字"**
- 模型永远只产出文本。所谓"执行命令""读文件"，都是它输出一段结构化文字（工具调用），由**外层程序**真正执行。
- 👉 启示：Agent 的"能力上限"由**你给它配多少工具**决定，不由模型决定。

**真相 2：模型不会"记得"，是每次被"重新喂"记忆**
- 我看起来记得你的偏好，其实是每次对话，系统把 GLOBAL.md 和日志**重新注入 prompt**。模型每次都是"第一次读到"。
- 👉 启示：记忆系统 = 检索 + 注入。做得好不好，决定 Agent 像不像"认识你很久"。

**真相 3："思考"可以被工程放大**
- 模型一次输出容易出错。但让它**多步走、自我检查、分角色辩论**，正确率大幅提升。
- 👉 启示：这是强化思维的**核心战场**，下面详讲。

## 第三部分：强化模型思维的 6 大引擎（可落地代码思路）

### 引擎 1：ReAct —— 思考与行动交替（基础）

```
Thought: 我需要先知道当前目录有什么
Action: shell("ls")
Observation: server.py  core/  web/
Thought: 有 server.py，我去读它的端口配置
Action: shell("grep PORT server.py")
Observation: PORT = 8765
Thought: 好，现在我知道了，可以回答
Answer: 服务运行在 8765 端口
```
**效果**：让模型"边查边想"，而不是凭空猜。
**Black God 现状**：agent_kernel.py 已有雏形 ✅

---

### 引擎 2：Reflexion —— 自我反思重做（提升正确率）

```python
# 伪代码
answer = model.solve(task)
critique = model.ask(f"批判这个答案，找出错误：{answer}")
if critique.has_problems:
    answer = model.solve(task, hint=critique)  # 带着批判重做
```
**效果**：模型当自己的审稿人，把错误答案修正。复杂任务正确率可提升 20-40%。
**Black God 现状**：缺，建议补 ⬅️ **强化重点**

---

### 引擎 3：Plan-and-Execute —— 先规划后执行（处理复杂任务）

```
1. Planner 把大任务拆成步骤清单：[步骤1, 步骤2, 步骤3...]
2. Executor 逐个执行，每步可调工具
3. 每步完成后检查：是否需要调整后续计划？
```
**效果**：避免"走一步看一步"迷路，适合多步骤复杂任务。
**Black God 现状**：task_controller.py 有部分 ✅，可强化

---

### 引擎 4：多智能体辩论 —— 集体智慧（最强思维提升）

```
同一个模型，扮演多个角色：
  🧠 规划者：提方案
  ⚙️ 执行者：落地实现
  🔍 批判者：挑毛病、找漏洞
  ⚖️ 裁判：综合各方，给最终答案

它们互相质疑 2-3 轮 → 答案质量远超单模型
```
**效果**：用"内部辩论"逼出更深的思考。难题尤其有效。
**Black God 现状**：葫芦娃/6引擎是这个思路 ✅，但缺真正的"辩论循环"

---

### 引擎 5：向量记忆 —— 让 Agent 真正"学习"

```
历史经验 → 转成向量 → 存向量库
新问题来了 → 检索"我以前遇到过类似的吗" → 把相关经验注入 prompt
```
**效果**：Agent 越用越聪明，能复用过去的成功经验。
**Black God 现状**：缺（现在是关键词搜索）⬅️ **可升级**

---

### 引擎 6：自我进化 —— 缺能力就自己造

```
任务失败 → 分析"缺什么能力" → 自动生成新 SKILL.md → 下次能用
```
**效果**：Agent 能力边界自动扩张。
**Black God 现状**：skill_index.py + skill-creator 有 ✅
