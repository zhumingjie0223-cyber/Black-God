# 神枢执行脑上线（GCP 版）—— 给神枢装上「真沙箱的手」

已备好、已验过：worker 侧 `execRemote` + agent 环 `⟨工具:exec｜命令⟩` + `⟨召唤:exec｜命令⟩`；
服务器侧 `server/exec_brain.py`（纯标准库、token 门、端到端真跑验证通过）。
接上只差一步：你那台 GCP 的**外部 IP** + 一个你自定的 **token**。

---

## 一、GCP 建机（控制台点几下）
Compute Engine → 创建实例：机型 **e2-small**、地区 **asia-east1(台湾)** 或 **asia-southeast1(新加坡)**、
系统 **Ubuntu 22.04**、勾**允许 HTTP/HTTPS**。记下**外部 IP**。
防火墙再放行执行脑端口（默认 8765）：VPC 网络 → 防火墙 → 允许 tcp:8765（来源可先 0.0.0.0/0，靠 token 守）。

## 二、在这台机器的 SSH 里，起执行脑
先想一个只有你知道的 token（随便一串，越长越好），下面用 `换成你的TOKEN` 占位：

```bash
sudo apt-get update -y && sudo apt-get install -y python3 git
git clone https://github.com/zhumingjie0223-cyber/Black-God.git   # 私库需你的 GitHub 凭证；或用下面的 scp/粘贴法
cd Black-God/server

# 常驻后台跑（token 必须设，否则拒绝启动）
NEXUS_EXEC_TOKEN=换成你的TOKEN nohup python3 exec_brain.py 8765 >/tmp/exec_brain.log 2>&1 &

# 自检：本机应答 health
curl -s http://127.0.0.1:8765/health
```

> 拉不动私库？把 `server/exec_brain.py` 一个文件 scp 上去，或 `nano exec_brain.py` 粘贴内容即可——它零外部依赖。
> 想开机自启/更稳，用 systemd（见文末）。

## 三、在 Cloudflare Worker 配两条 secret
在 `web/nexus-do/` 下：
```bash
npx wrangler secret put NEXUS_EXEC_URL      # 填 http://你的外部IP:8765
npx wrangler secret put NEXUS_EXEC_TOKEN     # 填 换成你的TOKEN（与服务器一致）
```
（或在 CF 面板 → 该 Worker → Settings → Variables 里加这两条，标为 Secret，然后 Deploy。）

## 四、验通
配好后，在神枢对话里说一句需要动手的（例：「在服务器上跑 `uname -a` 看看」）。
她会用 `⟨工具:exec｜uname -a⟩` 真跑，把真实输出带回来。没接通时她会**如实说未接入**，绝不假装。

## 附：systemd 常驻（更稳，推荐）
```bash
sudo tee /etc/systemd/system/shenshu-exec.service >/dev/null <<'UNIT'
[Unit]
Description=Shenshu Exec Brain
After=network.target
[Service]
Environment=NEXUS_EXEC_TOKEN=换成你的TOKEN
WorkingDirectory=/root/Black-God/server
ExecStart=/usr/bin/python3 /root/Black-God/server/exec_brain.py 8765
Restart=always
[Install]
WantedBy=multi-user.target
UNIT
sudo systemctl daemon-reload && sudo systemctl enable --now shenshu-exec
```

## 安全（安全工程师叮嘱）
- token 是唯一的门：越长越随机越好；不要发到任何公开处。
- 想更严：防火墙来源只放行 Cloudflare 的出口段，或给执行脑套一层 HTTPS（Caddy/nginx 反代）。
- 这台机器上跑的命令有真实权限——只服务主人自己或已授权的系统（铁律）。
- 红线机器 `47.93.100.200` / 乡村振兴：不碰。
