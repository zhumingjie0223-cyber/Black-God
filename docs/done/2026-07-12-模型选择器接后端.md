## 改动清单

**核心接线(nexus_do.core.mjs)**
- 新增纯函数 `pickBrainOrder(tier, model)`(放 pickTier 旁,确定性可测):
  - `cf-llama` → `['cf']`(用户明确只要免费内置,只用 CF);
  - `strong` → `['gateway', 'cf']`(网关优先,挂了仍诚实兜底 CF,不假装成功);
  - `auto`(含缺省) → light 先 CF 后网关 / heavy 网关优先后 CF(原智能路由行为)。
- `callBrain` 的后端尝试顺序改由 `pickBrainOrder(tier, opts.model)` 决定,
  取代原来写死的 `tier === 'light' ? [tryCF, tryGateway] : [tryGateway, tryCF]`。
- `handleTalk(text, request, capsIn, model)` 新增 model 参数,brainModel=model||'auto',
  透传给 agentic 分支的 runAgentLoop 和单发分支的 callBrain 的 opts。
  (runAgentLoop 内部 callBrain 已透传 opts,无需改它本身。)
- 三处 talk 入口都带上 model:HTTP `/talk`(b.model)、WebSocket(msg.model)、
  能力契约层 invokeCapability(params.model)。
- 未改 observe 内心独白(877)与自主守望(1196/1637)的 callBrain 调用——
  它们不传 model,保持 auto,省钱且合理(内心独白/守望不需用户模型偏好)。

**前端(index.html)**
- `Nexus.talk` 的 `/talk` 请求体 `{text, caps}` → `{text, caps, model: store.get('model')||'auto'}`。
  (`pickModel` 早已把选择存进 `store.set('model', id)`,此前只是没人读它发出去。)

**测试(selftest.mjs)**
- 新增 5 项 pickBrainOrder 断言:cf-llama 只用 CF / strong 网关优先兜底 /
  auto+light / auto+heavy / 缺省等同 auto(向后兼容)。全绿 88/88。
