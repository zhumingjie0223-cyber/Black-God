## 改动清单(逐函数)

**消除重复调用**
- router.py 原 `apply_route_to_session` 内嵌了一份 `find_entry_id` + 裸
  `subprocess.run(minis-config get models)`,和 router_v2 已加固的
  `_resolve_entry_id` 做同一件事却各写一套。现 router.py 直接 import 复用
  `_resolve_entry_id`——白拿它的进程内缓存、线程锁、returncode 校验,
  删掉一整段重复逻辑与一处 `models_data["value"]` 的裸下标(value 非 str 会崩)。

**消除复制粘贴**
- primary/sub 两段 `minis-config set` 写入代码逐字重复,收敛为
  `_write_session_model(config_key, entry_id)` 一处。顺带修一个漏判:
  原实现 `json.loads(r1.stdout) if r1.stdout else ...` 不看 returncode,
  子进程非零退出但恰好吐了可解析 JSON 时会被当成写入成功——现在
  returncode 非零一律判失败。

**接线死代码(最实质的一处)**
- `EMERGENCY_FALLBACK`(跨厂商紧急兜底路由)从定义之日起就没被任何函数
  调用过——primary/sub 模型解析不到时,原实现直接 `return {"error": ...}`
  就撂挑子,兜底对象形同虚设。现在 `apply_route_to_session` 在解析失败时
  自动降级到 EMERGENCY_FALLBACK 再试一次:
  - `_allow_fallback` 参数保证只降一层,兜底自己再失败就如实报错,绝不无限递归;
  - 成功兜底时结果带 `fell_back_to_emergency` + `fallback_reason`,调用方看得见发生过降级。

**健壮性**
- `score_complexity` 空描述/None 不再 `.lower()` 崩溃,退回纯步骤/文件数启发式。
