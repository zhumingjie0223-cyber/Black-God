# 任务:网关模型自动识别(权哥 2026-07-14 指令"不能自动联网识别网关模型呀")

## 问题

大脑网关配置里,网关地址、密钥、模型名三个都是手敲文本框——模型名要用户自己记住并敲对
(如 `sihan-max`)。全仓没有任何"填了网关就自动把它支持的模型拉出来"的能力;更糟的是后端
`callBrain` 里模型默认硬传 `'auto'`,而多数 OpenAI 兼容网关不认 `auto` 这个模型名,直接报错。

## 修复(前后端闭环)

**后端 `nexus_do.core.mjs`**
- 新增 `modelsEndpoint(base)`:从网关 base 推导标准 `/models` 端点(剥掉 `chat/completions`
  等尾巴、去尾斜杠、补 `/models`)。
- 新增 `probeModels(b)`:`GET {base}/models` 联网拉列表,兼容三种主流返回格式——
  OpenAI `{data:[{id}]}`、`{models:[...]}`、纯数组;结果去重;失败(无地址/4xx/空列表/连不上)
  返回中文错误提示,不抛异常。支持请求体直接传 `gateway_url`/`gateway_key`(填了还没保存也能识别),
  掩码 key 自动回退到已存密钥。
- 新增路由 `POST /config/models`(认证后)。
- `callBrain` 的网关分支:模型留空或为 `auto` 时,自动识别一次取第一个真实模型并缓存到
  `_auto_model`,不再硬传 `auto`;`setConfig` 换网关/换模型时清缓存,下次重新识别。

**前端 `index.html`**
- "大脑·模型"设置里,模型名一栏加「🔍 识别模型」按钮 + `<datalist>` 下拉:填了网关地址点一下
  即联网拉取,识别到的模型变成可下拉选(保留手填兜底),默认自动选第一个,并提示识别到几个。
- `Nexus.probeModels()` 网络方法 + `probeModels()` 交互函数。

## 实测(起本地 mock 网关真跑)

新增回归测试 `web/nexus-do/gateway.test.mjs`,已接入 `npm test`,**17 项全绿**:
- 端点推导 4 种(`/v1`、带尾斜杠、完整 `chat/completions`、`/messages`)
- probeModels 三种响应格式 + 去重 + 错误处理(空列表/无地址/401)+ 请求体传 key + 掩码回退
- **auto 闭环完整链路**:配了网关但模型留空 → 自动识别取到 `sihan-max`(不再硬传 auto)→
  成功回复 → 结果缓存 → 二次调用命中缓存不重复联网

`npm test` 汇总:主自测 85 + 网关识别 17 全通过;`sync-ui --check` 双副本一致;
`check-sync` 引擎层完全同步。

## 总结

权哥填个网关地址,神枢就自己上网问清楚"你这儿有哪些模型",列出来给他挑,挑不挑都行——
留空就自动替他定一个真实可用的,不再因为硬传 `auto` 被网关拒之门外。核心链路起 mock 网关真跑验证,
不是"应该能行"而是"实测能行"。
