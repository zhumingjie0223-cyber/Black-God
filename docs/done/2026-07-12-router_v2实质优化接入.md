## review 改动清单(逐函数)

**真安全洞(最重要的一个)**
- `evaluate_destructive_action`:原实现只要影子探测发现"目标为空"就把风险降级
  并豁免二次确认——连 CRITICAL(`rm -rf /`)也照降照放行。现在 CRITICAL 永不降级、
  永不豁免确认,降级通道只对 HIGH 开放;目标是符号链接时也不降级(判空看的是
  链接本身,真身可能非空);docstring 注明探测与执行间的时间窗限制,
  requires_confirmation=True 任何调用方不得绕过。

**v1 就潜伏的正则 bug(被新测试逮住)**
- fork 炸弹正则 `:\(\)\{.*\|.*&\};:` 匹配不了真实写法 `:(){ :|:& };:`
  (& 与 }; 之间有空格)——已改为允许各处空白。

**漏报修复**
- 删根检测只认 `rm -rf /` 一种写法,`rm -fr /`、`rm -r -f /`、
  `rm --recursive --force /`、`rm -rf //` 全漏——正则重写,旗标顺序/合并/长旗标全覆盖,
  同时保住"删子路径不误判成删根"的原始诉求(有回归测试钉死)。
- `git push origin main --force`(带远程分支)和裸 `git push -f` 原来都漏——补上,
  `--force-with-lease` 安全写法不误伤。

**健壮性**
- `assess_command_risk` 空命令/纯空白原来直接崩(None.lower()),现判 LOW。
- `run_advisory_panel` 空顾问名单原来会让线程池构造抛 ValueError;
  总超时原来把 TimeoutError 直接抛出去,丢光所有已完成顾问的意见——
  现在超时槽位记"缺席",已完成的照常带回。
- `call_model` 细分错误路径:子进程超时/非零退出/输出非 JSON/结构异常各给
  明确中文错误文案;临时文件清理挪进 finally(原实现异常路径会漏删)。
- `_resolve_entry_id` 补 returncode 校验;解析失败不再可能把 None 写进缓存
  毒化后续调用(只缓存有效值);加线程锁防会诊并行时缓存竞写;
  裸 except 收窄为具体异常清单。
- `wrap_compacted_summary` 幂等化:已包裹的摘要原样返回,防多轮压缩叠嵌套标记;
  空摘要返回空串。`is_compacted_summary` 识别标记派生自前缀常量,消除双写漂移;
  None 容错。

**性能**
- `sandbox_probe` 大目录判空从 `len(os.listdir())` 全量拉取改为 scandir 首项即断
  (百万文件目录不再卡死);增加 is_symlink 探测项。
- `aggregate_advisory` 全部顾问缺席时短路返回,省掉一次最贵的聚合模型空调用。

**类型注解/整洁**
- 全模块 `list`→`list[AdvisorSlot]`/`list[AdvisoryResult]`、`dict`→`dict[str, Any]`、
  缓存 `dict[str, str]`;函数内 import(tempfile/os/re/concurrent.futures)全部上提;
  错误截断长度收敛为常量。

## 接入 router.py 主流程

`route_task` 扩展(老调用方式行为完全不变):
- `command=`:高危命令先过安全门,需确认的直接返回 blocked 决策,
  一个配置写入都不花;
- `history_summary=`:摘要经 CompactionGuard 包裹后随决策返回;
- `advisory=`:None 时 HIGH/MAX 复杂度自动开会诊(顾问=低两档的 Primary,
  聚合=执行档 Primary,build_advisors_for 组团),顾问全缺席不阻塞主流程。
- `__main__` 自测新增安全门/防污染离线用例,不烧任何模型调用。
