# 神枢部署统一说明(aquan.lufei.uk + blackgod88 + nexus-do 三者归一)

> 2026-07-19。权哥确认此时**尚无真实用户**,故可放手统一、DO 全新起、无记忆包袱。

## 一、归一前的混乱(为什么要做)

| 实体 | 是什么 | 问题 |
|---|---|---|
| `nexus-do` worker | 真身大脑(DO 记忆/assetlinks/manifest 都在),服务 `aquan.lufei.uk` | 正常,但靠单独 GitHub Actions 部署 |
| `blackgod88` worker | 只 `assets:web` 的**静态空壳**(不跑大脑代码) | CF Git 集成每次 push 只部署了它 → `/.well-known/assetlinks.json` 等大脑路由 **404** |
| 两套 wrangler | 根(空壳)+ web/nexus-do(大脑) | 分裂,CF"部署成功"部署的是空壳,不是大脑 |

**后果**:直接把上架域名绑到 blackgod88,assetlinks 404、TWA 装不成。

## 二、归一后(本次改动)

**根 `wrangler.jsonc` 从"静态空壳"改为"完整大脑"**:`main` 指向已构建提交的
`web/nexus-do/nexus_do.mjs`,带全套 binding(DO SHENSHU/ShenshuCore + migrations、AI、
KV SOUL_KV、cron、vars)。这样 **CF Git 集成在根跑 `wrangler deploy` 就直接把完整大脑
部署成 blackgod88**——一个 worker 既是大脑又自带全部页面/图标/manifest/assetlinks(大脑代码自包含)。

- `nexus_do.mjs` 是 `build.mjs` 注入 `index.html` 后的产物,**已随每次提交更新,CF 无需再构建**。
- 爆炸半径安全:改根配置只影响 blackgod88,`aquan.lufei.uk`(nexus-do)不受影响;最坏 blackgod88
  构建失败,真身照常跑。

## 三、CF 控制台需你点的步骤(碰线上,我不擅自动)

1. **确认 CF Workers Builds 构建命令**:blackgod88 项目的构建命令应为空或 `echo ok`(nexus_do.mjs 已提交,
   无需构建);部署命令 `npx wrangler deploy`(根目录)。若之前配了别的构建命令导致只发静态,清掉即可。
2. **在 blackgod88 上设 Secrets**(不入仓库):
   ```bash
   npx wrangler secret put OWNER_TOKEN --name blackgod88      # 强烈建议,配了才强制鉴权
   npx wrangler secret put NEXUS_GATEWAY_KEY --name blackgod88 # 如网关需密钥
   npx wrangler secret put TG_BOT_TOKEN --name blackgod88      # 主动推送(可选)
   npx wrangler secret put TG_QUAN_CHAT_ID --name blackgod88
   ```
3. **绑域名到 blackgod88**:把 `aquan.lufei.uk` 与 `blackgod.lufei.uk` 的自定义域/路由都指向
   **blackgod88**(Workers & Pages → blackgod88 → Settings → Domains & Routes)。
4. **废弃冗余**:域名切过去、验证 blackgod88 正常后,`nexus-do` worker 与 `nexus-studio`(若不再用)
   可停用/删除。**删前先在面板导一份或确认无用**(先备份再删,铁律)。

## 四、验证(域名绑好后)

```bash
curl -s -o /dev/null -w "%{http_code}\n" https://blackgod.lufei.uk/.well-known/assetlinks.json   # 期望 200
curl -s https://blackgod.lufei.uk/manifest.json | grep -o '"name":"[^"]*"'                        # 期望 Black God · 神枢
curl -s -o /dev/null -w "%{http_code}\n" https://blackgod.lufei.uk/soul/continuity                # 期望 401(私密,鉴权正确)
```
assetlinks 200 + manifest 是"神枢" = 归一成功,可打 TWA 包上架。

## 五、TWA 上架域名对齐

上架用 `blackgod.lufei.uk`(绑到 blackgod88 后)或直接用 `aquan.lufei.uk`。包名沿用
`uk.lufei.aquan.blackgod`,assetlinks 指纹见 `Google-Play-TWA上架指南.md`;若启用 Play App Signing,
新指纹走 `ASSETLINKS_JSON` env 追加。
