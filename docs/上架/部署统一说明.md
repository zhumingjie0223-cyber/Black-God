| 实体 | 是什么 | 问题 |

## 二、走过的弯路(如实记录)

曾尝试把根 `wrangler.jsonc` 改成"完整大脑",让 CF Git 集成把 blackgod88 部署成大脑。
**结果 CF 构建失败**——**静态资源型 worker 不能就地转成带 Durable Object 的脚本 worker**。
故**撤回**该改动,blackgod88 保持静态壳原样(其构建随之恢复正常)。

## 三、最终方案:唯一真身 aquan.lufei.uk(安全、零折腾)

`aquan.lufei.uk`(nexus-do worker)本就是完整大脑,已验证:
```
/.well-known/assetlinks.json → 200 ✅
/manifest.json name          → Black God · 神枢 ✅
/soul/continuity             → 401（私密,鉴权正确）✅
/privacy 数据主权导出条款      → 在 ✅
```
**上架 TWA、主域一律用 `aquan.lufei.uk`**,包名 `uk.lufei.aquan.blackgod`(与该域自洽)。不需要任何危险的 worker 改造。

## 四、CF 控制台可选收尾(你点或授权,不急)

1. **弃用 blackgod88 静态壳**:域名都在 aquan.lufei.uk 上,blackgod88 已无用,可在
   Workers & Pages → blackgod88 → 停用/删除(**删前确认无域名指向它**;先备份再删)。
   删掉后 CF Git 集成对 blackgod88 的部署也就不再有意义,可一并解除。
2. **nexus-do 部署方式**:继续走它自己的 `web/nexus-do/wrangler.jsonc`(GitHub Actions / 手动
   `cd web/nexus-do && npm run deploy`)。这是唯一真身的部署路径。
3. **Secrets 已在 nexus-do 上**(OWNER_TOKEN 等),无需迁移。
4.(可选)若将来想要 `blackgod.lufei.uk` 域名,直接在 CF 面板把它作为自定义域**绑到 nexus-do**
   (不是 blackgod88),assetlinks 立即可用。
