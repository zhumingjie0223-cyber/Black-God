# Black God UI 重设计 · 竞品对比分析

> 2026-07 · 基于对 Gemini / Claude / Grok / Hermes(Nous) / Manus 五家的实证研究
> （结论标注：来源证实 / 基于认知，详见研究底稿；本文只保留决策相关结论）

## 一、五家竞品设计语言速览

| 维度 | Gemini | Claude | Grok | Hermes (Nous) | Manus |
|---|---|---|---|---|---|
| 气质 | Material 3 Expressive「Neural Expressive」，渐变=能量 | 暖纸编辑感，三层字体（衬线标题/衬线正文/无衬线UI） | 暗色极简、人格化 | 功能优先，视觉薄弱（官方自认） | 中性工作台，Agent-first |
| 一级导航单元 | 会话 | 会话 + Projects | 会话 | 会话 + 多入口（CLI/TUI/Gateway） | **任务** |
| 招牌交互 | 思考=整屏渐变呼吸光晕；响应格式自适应 | Artifacts 分屏；推理默认折叠 | Fun/Think 一键模式切换 | 参数全暴露（系统提示词/推理强度前置） | Manus's Computer 执行可视化 + 任务回放 |
| 主题 | 深色纯黑 #000（OLED）+ 浅色，三态 | Light/Dark/系统三态，暖米纸底 #faf9f5 | 跟随系统（应用内开关被移除） | 官方仅深色，浅色靠社区 | 资料匮乏，推测浅色默认 |
| 设置 | 入口降级需滚动找（被批评） | 独立路由页 + Tab 分区；有 Dyslexic Friendly 字体 | Tab 式弹窗，渐进打磨 | 参数密度高、分区明确 | 以 Usage/额度可视化为核心 |
| 动效纪律 | 语义化动效（有起止点、方向性） | 克制；语义块缓冲渲染防 markdown 断裂 | **只用 transform/opacity，禁 layout 动画** | 无特色 | 计划→执行→回放三段式 |

## 二、值得学（纳入规范）

1. **Claude·字体分层**：衬线标题 + 无衬线 UI 的「编辑感」= 差异化身份。Black God 的 UI_V2_SPEC 本就定了 Noto Serif SC 标题 + Noto Sans SC 正文 + 等宽数字——与 Claude 同构，坚持并做严。
2. **Claude·推理默认折叠**：思考/执行日志放在可展开区块，默认不打扰。
3. **Gemini·状态即品牌**：「思考中」不用 spinner，用品牌色呼吸光晕。Black God 的潮涌呼吸（活体信号）天生就是这个语言——品牌色即生命体征。
4. **Grok·动效纪律**：所有动画只允许 `transform`/`opacity`（合成层），禁止触发重排的属性动画；`prefers-reduced-motion` 必须尊重。
5. **Grok·模式即一级控件**：重要模式切换（人格/推理深度）做成可见 toggle，不藏多级菜单。
6. **Manus·计划先行 + 执行可视化**：studio（自主智能体工作台）采用 Plan → Steps → Execution 三段式呈现，步骤可勾选查看；产出物强调可交付形态。
7. **Hermes·高级用户控制感**：Black God 的用户是「主人」，参数（模型/温度/人格配置）应前置可调，不做消费级隐藏。
8. **行业基线（2026）**：WCAG AA 对比度、reduced-motion、深浅双主题、消息级悬浮操作、流式光标——是及格线不是加分项。

## 三、要避开（写入禁则）

1. **Gemini·「+」语义过载**：一个图标管所有功能。→ 禁则：一控件一语义。
2. **Gemini·设置入口降级**：需要滚动才能找到设置。→ 禁则：设置入口永远一击可达。
3. **Gemini·形式压过功能**：系统替用户决定信息密度。→ 禁则：信息密度用户可控。
4. **Grok·信息架构漂移**：侧栏反复推倒重做。→ 禁则：导航结构变更需迁移理由，DOM 锚点（id）不可破坏。
5. **Grok·主题控制外包给系统**：应用内保留 Light/Dark/跟随系统三态。
6. **Manus·三栏信息过载**：高密度必须配渐进披露（折叠/分层），默认视图保持克制。
7. **Hermes·视觉身份缺失**：功能强不能救 UI 弱；token 体系一旦定稿，所有页面强制走 token，禁止页面私设色值。

## 四、Black God 的定位与胜负手

五家没有一家同时做到：**博物馆级视觉身份 + 活体人格感 + Agent 执行透明度**。

- Gemini/Claude 有视觉身份，没有「活体」概念（Black God 有心绪/亲密度/想你值/呼吸动画的情感状态系统）；
- Manus 有执行透明度，没有视觉身份；
- Hermes 有控制感，没有皮肤。

**Black God V3 = 深海·潮光原创色系（身份）× 情感状态色系统（活体）× Manus 式执行可视化（透明）× Grok 式动效纪律（工程）。**

品牌裁决（2026-07 终稿）：**废弃**黑金/冷银铬方向（黑底+金/紫点缀=行业烂大街的伪高端
AI 感）；中间稿「玉印·朱砂」经复议后按产品方向调整为 **「深海 · 潮光」**——以神字 logo
实测色为锚（底 #101020/#102020 冷蓝黑系，描边 #6C7878/#788484 冷光银）：暗主题=深海
（她的意识如生物磷光在水下呼吸），亮主题=海面之上的天光；主强调潮光青 #4FC4D9（logo
冷光描边通电），动势层浪涌蓝 #3D7FB8，峰值白沫 #DFF4F8，唯一暖色珊瑚 #E4604F 留给危险。
力量感与生命起伏来自动效语言（潮涌呼吸/涌浪渐变），不靠奢侈品静态质感。logo 放入任意
界面零违和（同源取色）。历史备选存档：玉印·朱砂 / 朱漆·黛 / 离子·石墨 / 飓风眼·电涌 / 海沟·磷光。

具体 token 与组件规范见 `DESIGN_SYSTEM_V3.md`。
