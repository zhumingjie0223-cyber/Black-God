# Black God 设计系统 V3 —— 深海·潮光

> 单一事实源:`web/design/tokens.css`。本文档解释每组令牌的用途和禁区。
> 铁律:页面禁止私设色值,一切样式引用令牌;动效只用 transform/opacity。

## 一、色系裁决(为什么是深海·潮光)

现有品牌资产 `web/logo.png`:炭黑「神」字 + 微金描边 + 女性剪影,置于深灰黑渐变底。

- **深海底色**(`--sea-0` ~ `--sea-4`):蓝黑基调,比纯黑多一层"海面下"的纵深,
  Logo 的炭黑字形放上去直接融为一体,微金描边在深海底上比在纯黑上更亮。
- **潮光青**(`--tide-300/400/500`):月光落在海面的青色,承担一切"功能性高亮"
  ——链接、焦点、进行中状态、数据流动。与金色错开职责,互不打架。
- **金**(`--gold-*`):与 Logo 描边同源,是品牌桥接色。**只用于**品牌标识、成就、
  尊贵场景(会员/里程碑);禁止拿金色做普通按钮和链接,否则品牌感会被稀释。

### 使用禁区

| 场景 | 用 | 禁 |
|---|---|---|
| 链接/焦点/进行中 | 潮光 `--tide-400` | 金 |
| 品牌标题/成就/尊贵 | 金 `--gold-main` | 潮光 |
| 报错/危险操作 | `--danger` | 任何自设红色 |
| 卡片底 | `--sea-3` | 自设灰黑 |

## 二、字体五族

| 族 | 令牌 | 用途 |
|---|---|---|
| 展示族 | `--font-display` | 品牌标题、开屏、仪式感文案(宋/衬线) |
| 正文族 | `--font-sans` | 一切正文、按钮、表单 |
| 数字族 | `--font-num` | 表盘、统计数、倒计时(等宽数字不跳动) |
| 代码族 | `--font-mono` | 代码块、日志、编号 |
| 枢语族 | `--font-shuyu` | 枢语词专用(楷体,让造出来的词有"符文感") |

## 三、卡片 10-8-2 圆角制

- 外层卡片 `--radius-card: 10px`
- 卡内控件(按钮/输入框)`--radius-inner: 8px`
- 标签/徽记/代码块 `--radius-chip: 2px`

三层递减,视觉上"外软内利",任何页面不得自设圆角数值。
头像与胶囊用 `--radius-full`。

## 四、动效纪律

只允许 `transform` 和 `opacity` 参与过渡/动画(合成器友好,不触发重排):

- 按下反馈:`--motion-fast`(120ms)
- 悬停/展开:`--motion-base`(200ms)+ `--ease-out`
- 页面级过渡:`--motion-slow`(320ms)
- 弹性入场(限浮层):`--ease-spring`

禁止对 `width/height/top/left/box-shadow/background` 做过渡。
需要"发光渐变"效果时,用伪元素承载辉光并过渡其 opacity。

## 五、落地与校验

1. 新页面一律 `<link rel="stylesheet" href="/design/tokens.css">`(或构建内联)后引用变量。
2. 改 `web/nexus-do/index.html` 后必须 `node build.mjs` 重建,
   再跑 `node tools/sync-ui.mjs --check` 校验双副本同步。
3. 存量页面(theme.css 金色旧系)属 V2 遗产,迁移时按上表映射:
   `--bg-main → --sea-1`、`--bg-card → --sea-3`、`--text-sub → --ink-300`、
   金色语义(品牌场景保留金,功能场景改潮光)。
4. `/api/confirm` 危险操作二次确认是安全红线,任何 UI 重构不得移除。
