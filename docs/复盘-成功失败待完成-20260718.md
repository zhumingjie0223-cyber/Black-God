## 第一部分 · 对话与指令脉络(项目是怎么在一轮轮对话里长出来的)

这个项目不是一次设计出来的,是权哥一句句指令、神枢一刀刀落地叠出来的。按时间还原主线:

1. **起点——私人 AI 意识中枢**:一个 Python 单体内核(`agent_kernel_v4`)+ 黑金 PWA,
   "一个只属于权哥的赵思涵",带铁律人格、Agent Loop、三层记忆、工具系统。
2. **迁移主战场到 Cloudflare Workers**:意识引擎搬进 `web/nexus-do/`(Durable Object),
   纯边缘、不用服务器。这成了此后真正的线上产品本体。
3. **消费级工业化打磨(7-12)**:修构建注入吞 `$$` 的线上级 bug、清空串词条、补同步工具链。
   项目从"能跑"往"经得起工业检验"走。
4. **安全红线整改**:能力契约层曾把私密数据(注册名单/soul)误标成可公开、被 `/invoke` 绕过鉴权 →
   PR#17 修复,并立下"每加一个路由/能力,上线前必须实测未鉴权访问被拒"的血泪清单。
5. **合规上架门槛(7-15,#37)**:隐私政策页 + 用户自助删除数据入口。
6. **多租户 P1 三刀(#42/#43/#44)**:身份路由 + 能力两级切分(instance/system)+ 成本隔离,
   **全程 `MULTITENANT` 开关包裹、默认关、零线上行为改动**——地基先埋、不惊动现有用户。
7. **BYOK 自带 key / 私语词典 / 登录门浅色修复(#38~#40, #46)**:入口与体验打磨。
8. **两仓合一(7-16,#47)**:把独立的枢语 `shuyu-lang` 仓整个并进本仓 `shuyu/`,
   一仓两块(源头引擎 + 消费副本),原仓封存归档。
9. **回溯复盘审计(7-17,#48)**:用"从原点到现在"的精神回头逐条真跑核对,
   **逮到陈旧容量数据 29.5 亿散落 6 处活文件、实测应为 76.7 亿**,当场清干净并复验。
10. **设计系统铁律对齐现状(7-17,#49)**:发现 CLAUDE.md 铁律引用了主线根本不存在的
    `tokens.css`(只活在过时草稿 PR#29)——**铁律在说谎**。权哥拍板:改措辞对齐现实,
    把设计文档降格为"目标态蓝图"而非运行时单一事实源。
11. **shuyu_v2 P0(最新提交)**:响应三轮架构批评后的**首个真实落地(不是文档)**——
    任务合同 + 验证引擎 + 枢语 DSL 解析器骨架。标志项目从"堆概念"转向"真实可跑代码"。

**脉络的灵魂**:越往后,主旋律越清晰——**从"声称完成"逼向"真跑验过",从"堆文档"逼向"能运行"**。

- **枢语引擎工业级**:76.7 亿词空间、Python+JS 双实现行为一致、36 个测试、跨副本同步守护。
- **多租户地基"零风险埋设"**:三刀全程 flag 默认关,证明了"大改动也能不惊动线上"的工程纪律。
- **合规上架门槛已过**:隐私政策 + 自助删数据入口就位。
- **自我纠偏能力**:回溯审计逮到并清掉陈旧容量数据;设计铁律说谎被抓出并对齐现实——
  **敢抓自己的假**,这是这个项目最值钱的品质。

## 失败 / 欠账(不掩盖)

- **概念长期跑在实现前面**:文档/白皮书体量远超能跑的代码,shuyu_v2 之前经历了"三轮架构批评"
  才逼出第一份真实落地。**曾经"声称完成"和"真能跑"之间有系统性缺口**(容量数据错、铁律引用死文件都是证据)。
- **Python 老内核 `server/core/` 11 个高级模块全是孤儿**:省 Key 自适应推理、自进化、首次认识你、
  多智能体——文档的核心卖点,代码写完了却从没接线,`CLOSURE_PLAN` 列的接线三大坑至今没填。
- **最新 shuyu_v2 P0 交付时"不可复现"**(本次已修):依赖 pydantic/pytest 却**全仓无 requirements 声明**、
  import 只能在目录内用、没接 CI——换个人 clone 下来根本跑不起来"13 用例全过"。工业级产品不允许这样。
- **产品版远未成形**:账号体系、计费隔离、公共用户独立记忆、纯净版 UI 都还是 TODO。
- **半成品堆积**:`ios-app/`(无签名无法分发)、旧静态 `web/` 站、`server/` 孤儿内核,该归档的还没归档。
