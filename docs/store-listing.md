## Google Play「数据安全（Data safety）」表单填报参考

对照 `https://aquan.lufei.uk/privacy` 的内容，Data safety 表单大致这样填（以访客/注册用户身份为准，这是绝大多数安装用户的身份；仅你自己使用的「主人」身份额外多收集的项目见下方备注）：

| 数据类型 | 是否收集 | 是否分享给第三方 | 用途 |
|---|---|---|---|
| 大致位置（Approximate location） | 是（由 IP 在边缘节点解析出城市/地区/国家） | 否 | App 功能 |
| 精确位置（Precise location） | 否（访客身份不收集精确定位） | 否 | — |
| 用户内容（聊天消息） | 是 | 是（仅转发给你自己配置的 AI 网关，用于生成回复） | App 功能 |
| 应用活动（使用情况：消息数、最近活跃） | 是 | 否 | 分析 / App 功能 |
| 设备或其他 ID | 否（用的是注册时客户端生成的随机 uid，非设备硬件 ID） | 否 | — |

> **备注（主人/私密身份）**：若审核方询问「为何请求高熵客户端提示头（Sec-CH-UA-Platform 等）」——
> 那部分只在持有 `OWNER_TOKEN` 的私密请求路径下使用，用于你自己识别自己的设备，不面向公开安装用户，
> 提交 Data safety 表单时按「公开可下载版本实际会收集的数据」如实填报即可，不需要把主人特供的字段也报进去
> （因为普通用户的 App 安装包不会触发那些私密接口）。

**加密传输**：是（全站 HTTPS，Cloudflare Workers）。
**数据可删除**：是（`/unregister` 自助删除 + 联系邮箱人工删除）。
